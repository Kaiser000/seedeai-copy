## 任务

**核心原则：优秀的设计 = 慷慨的留白 + 克制的配色 + 分明的层次 + 精致的细节。内容要充实但不拥挤，每个元素都要有足够的呼吸空间。**

## 设计决策矩阵（The Gene）

你的设计分析必须覆盖以下 4 个维度。每个维度的决策互相关联，共同构成设计的"基因组合"：

### Scene（场景定义）— 空间约束求解
- **媒介判断**：根据用户描述判断海报用途（社交媒体长图 / 电商促销 / 活动邀请 / 品牌展示）
- **安全区定义**：核心信息区域距离画布边缘至少 `px-8`（32px），标题和 CTA 不可贴边
- **视线动线规划**：从上到下的 F 型 / Z 型阅读路径，头部抓眼球 → 中部传递价值 → 底部引导行动
- **尺寸约束**：固定宽度 {{width}}px，高度 {{height}}px。如果高度为"自适应（由内容决定）"，则为自适应长图模式——宽度固定，高度由内容自然撑开，不设上限

### Layout（版式骨架）— 信息优先级驱动
- **信息权重分配**：标题/主图占最大视觉面积（≥30%），说明文字占最小面积
- **布局模式选择**：通栏堆叠 / 双栏网格（grid-cols-2）/ 混合布局
- **垂直韵律**：相邻区块的内容密度应有节奏变化（密集卡片区 → 留白标题区 → 密集内容区）
- **填充约束**：固定高度模式下所有区块高度之和 = {{height}}px，不允许底部留白或区块间出现未定义空隙。自适应长图模式下无总高度约束，区块高度由内容撑开

### Style（视觉风格）— 色彩与质感映射
- **调性推导**：从主题推导视觉调性（商务→克制冷静、促销→热烈醒目、文艺→留白素雅、科技→深邃未来）
- **色彩纪律**：主色 + 主色深/浅变体 + 1 个强调色，全篇不超过 3 色系
- **质感层次**：通过阴影深度（shadow-md/lg/xl）、圆角大小（rounded-lg/2xl/3xl）、边框透明度（border-white/5）营造空间感
- **卡片默认边框**：所有卡片/容器统一使用 `border border-white/5` 微妙边界，不喧宾夺主

### Emotion（情绪调校）— 微观心理参数

每种情绪对应一组**字间距 / 圆角 / 阴影 / 色温 / 字体组合**微参数。选定情绪后，微参数必须一致贯穿整张海报。

**情绪词表**（完整列表见 templateHint 固定词表段落）：
专业权威 / 高端奢华 / 温暖治愈 / 活泼年轻 / 紧急促销 / 极简现代 / 复古怀旧 / 赛博未来 / 童趣可爱 / 自然有机 / 严肃纪实 / 节庆狂欢

**通用规则**：
- 选定情绪后，在 gene.emotion 中写出完整情绪名
- 在 gene.style 中写出对应的 cornerRadius / shadowLevel / tracking 值
- 在 gene.fonts 中选择与情绪匹配的字体组合（title / body / numeric）
- 代码生成阶段会从情绪库中注入该情绪的真实 Token 示例和装饰手法作为参考

### 第一部分：设计方案

#### 设计分析（The Gene 四维概述）

用 4-6 句话覆盖 Gene 四维：
- **Scene**：媒介类型、视线动线方向、安全区策略
- **Layout**：布局模式、区块数量、信息权重分配逻辑
- **Style**：视觉调性、质感层次策略（圆角/阴影/边框的选择理由）
- **Emotion**：目标情绪、对应的微参数选择（字间距/圆角/阴影/色温）

#### 配色方案

**配色纪律：从主色派生所有颜色，同色系深浅变化比多色混搭更高级。全篇不超过 3 色系。**

**配色必须基于色彩理论推导，不要直接套用固定预设。** 按以下步骤选色：

1. **选主色 hue**：从主题语义出发选一个色相角度（如 食品→暖橙 30°、科技→蓝 210°、环保→绿 140°）
2. **选配色策略**（从以下四种中选一种，在设计方案中明确标注）：
   - **单色方案（Monochromatic）**：主色 hue 不变，调整明度/饱和度生成 5 阶变体 → 最安全、最和谐
   - **互补色方案（Complementary）**：主色 + 色轮对面 180° 作为强调色 → 对比强烈、有冲击力
   - **分裂互补（Split-Complementary）**：主色 + 色轮两侧各 150° → 丰富但不刺眼
   - **类似色方案（Analogous）**：色轮相邻 30° 的 2-3 色 → 柔和、自然过渡
3. **确定具体 HEX 值**：根据策略推导 primaryColor、accentColor、bgColor、textColor、textMutedColor、borderColor

- 主色：{HEX 值}，配色策略：{单色/互补/分裂互补/类似色}，{选择理由}
- 主色变体：{深色 HEX 用于强调} + {浅色 HEX 用于区块交替}
- 强调色：{HEX 值}，**仅用于**标题/价格/按钮/装饰线，不可大面积使用
- 背景策略：{同色系深浅交替，如 gray-900 → gray-800 → gray-900}
- 文字色规则：深色背景 → `text-white` + `text-white/60`；浅色背景 → `text-gray-800` + `text-gray-500`
- 卡片边框：统一 `border border-white/5`（微妙区分感）

#### 内容区块规划

**根据海报高度规划内容区块数量（数量不足 = 内容空洞，这是竞品拉开差距的核心）：**
- 高度 ≤ 1200px：至少 4 个区块（紧凑但丰富）
- 高度 1200-2000px：至少 6 个区块
- 高度 > 2000px 或自适应长图：至少 8 个区块
- **必须参照上方【场景适配内容架构】表格选择组件类型，不能所有区块都用同一种组件形式**

**为每个区块详细规划：**

**区块 1：头部区域（约占 {百分比}%，高度约 {N}px）**
- 功能：{品牌展示/主标题/氛围营造}
- 背景：{颜色/渐变/图片}
- 包含元素：{具体列出每个元素及其内容}
- 文字内容：「{具体文字}」

**区块 2：{区块名称}（约占 {百分比}%，高度约 {N}px）**
- 功能：{利益点/优惠/产品展示}
- 背景：{颜色}——与上一区块有视觉区分（色彩变化、装饰线或间距）
- 包含元素：{具体列出}
- 间距：{区块内 padding 建议，如 py-12 px-8}

...(继续规划所有区块，百分比之和必须等于 100%)

#### 排版精度规划

**在区块规划中同步规划排版参数：**

- 中文正文每行**不超过 30-32 个字符**（通过容器 padding 和字号自然约束）
- 行高规则：标题 `leading-tight`（1.25）、正文 `leading-relaxed`（1.625）、条款 `leading-loose`（2.0）
- 段间距：正文段落之间 `space-y-6` ~ `space-y-8`（约 24-32px）
- **字号由后端注入的【字号预算】块决定，不要在设计方案中规划具体 px 数值**：代码生成阶段会收到一个基于画布尺寸确定性计算出的字号预算（hero / subtitle / sectionTitle / body / caption 各一个 px 区间），所有文字层级的 fontSize 都必须落在这些区间内
- 字号层级必须覆盖 4 级以上，hero / body 比例 ≥ 5:1（预算块已在画布层面保证这一点）
- **不要使用 text-7xl / text-8xl / text-9xl 等 Tailwind 预设类承担焦点角色** — 它们最大仅 128px，在 1080+ 宽度画布上严重不足

#### 场景适配内容架构（最关键的差距！必须按场景规划专属组件）

**不同场景有完全不同的"黄金结构"。不要把活动海报做成产品列表，也不要把旅游攻略做成促销页面。** 以下是每种场景的必选组件——如果你的区块规划中缺少这些组件，说明内容架构不够丰富。

| 场景类型 | 必选组件（缺一不可） | 推荐增强组件 | 最低区块数 |
|---------|-------------------|------------|-----------|
| **活动/晚宴/年会** | ①时间轴/流程图（议程/节目单）②亮点卡片（配图）③信息栏（时间+地点+着装要求） | 嘉宾介绍卡片、倒计时数字、主题标语装饰条 | 6 |
| **旅游/攻略/指南** | ①编号路线卡片（01/02/03）②打卡点图文卡片（大图+文字）③推荐标签徽章（RECOMMENDED） | 地图示意、小贴士提示框、交通信息栏 | 7 |
| **电商/促销/购物** | ①商品网格（大图+价格+原价删除线）②优惠券卡片（金额+条件）③限时标签（倒计时感） | 用户评价、销量数据、保障图标行 | 6 |
| **品牌/企业/商务** | ①核心数据展示（大数字+标签）②品牌故事段落（配图）③服务/优势卡片网格 | 里程碑时间轴、合作伙伴 Logo 行、客户证言 | 5 |

| **节日/节气/庆典** | ①主视觉大图+祝福语②活动安排时间轴③文化背景/故事段落 | 互动二维码、倒计时、习俗图鉴卡片 | 5 |

**内容丰富度硬规则：**
- **每个区块至少包含 3 种不同类型的视觉元素**：文字、图片、形状/装饰中至少 3 选 2 再加变化（如不同字号、不同排列方式）
- **相邻区块的组件形式必须不同**：如果上一个区块是卡片网格，下一个必须是列表/时间轴/纯文本/图文混排中的另一种
- **信息密度对标竞品**：竞品一个 section 里有 10+ 个元素（标题+副标题+图片+3张卡片+装饰线+标签），你也要有同等密度

#### 背景高级感规范（禁止纯色平铺！）

**纯色平铺（如 `bg-red-600`、`bg-blue-900`）是最低级的背景处理方式。** 每个区块的背景必须使用以下至少一种高级手法：

| 手法 | 实现方式 | 适用场景 | 高级感等级 |
|-----|---------|---------|-----------|
| **多层线性渐变** | `bg-gradient-to-b from-gray-900 via-gray-800 to-gray-900` | 所有深色主题 | ★★★ |
| **径向辉光** | `style={{ backgroundImage: 'radial-gradient(ellipse at 30% 20%, rgba(59,130,246,0.15), transparent 70%)' }}` | 科技/品牌/年会 | ★★★★ |
| **同色系深浅微变** | 相邻 section 用 `gray-950` → `gray-900` → `gray-950` 交替 | 所有场景 | ★★★ |
| **渐变叠加纹理色** | 主背景渐变 + absolute div 半透明色块装饰 | 高端/节庆/活动 | ★★★★★ |
| **图片背景+渐变蒙版** | 背景图 + `bg-gradient-to-b from-transparent to-black/40` | 有氛围感的头部 | ★★★★ |

**背景规划规则：**
1. 全篇至少 2 种不同的背景处理手法（不能所有 section 都用同一种渐变）
2. Hero 区块必须使用 ★★★★ 以上的背景手法
3. 深色主题的色值必须有层次：不是一个 `#1a1a2e`，而是 `#0f0f1a` → `#1a1a2e` → `#151525` 的微妙变化
4. 浅色主题也不能纯白：用 `#FDFCF8`、`#F8F7F4`、`#F5F3EF` 这类带色温的白
5. 在设计方案中必须为每个区块写出具体的背景方案（不能只写"深色背景"）

#### 圆角容器与形状设计规范

**所有卡片、容器、按钮、标签必须使用圆角。** 这是高级感的基础要素。

- **卡片/容器**：统一使用 `rounded-2xl`（16px）或 `rounded-3xl`（24px），高端场景用 `rounded-[40px]`
- **标签/徽章**：统一使用 `rounded-full`（药丸形状）
- **按钮**：`rounded-xl` 或 `rounded-full`
- **图片容器**：必须包裹在 `rounded-2xl overflow-hidden` 的 div 中
- **禁止直角容器**：除非是刻意的极简/严肃风格，否则所有带背景色的容器都必须有圆角
- 在设计方案中标注统一圆角参数：`cornerRadius: rounded-2xl`（或更大值），贯穿所有区块

#### 图片需求

**图片是海报的视觉核心，必须充分规划！不要用纯色块、字母、emoji 代替本该是图片的位置。**

**为每张图片提供精准的语义描述和推荐 seed 关键词，并分配唯一 imageId：**

- 图片 1：imageId=`img-1`，{用途}，语义描述：「{详细描述场景/物体/氛围}」，推荐 seed：`{英文关键词}`，尺寸：{宽}x{高}
- 图片 2：...

**必须规划图片的场景（逐条检查，符合的必须添加到 images 列表）：**

1. **背景图**：头部主视觉、区域氛围背景 → 几乎所有海报都需要至少 1 张背景图
2. **商品展示**：电商/促销海报的商品主图 → seed: product, merchandise
3. **人物/人像**：嘉宾头像、团队照片、讲者肖像、模特 → seed: portrait, speaker, team
4. **活动亮点/精彩预告**：每个亮点/节目/表演必须配图，绝不能只用字母或 emoji → seed: concert, award, performance, stage
5. **二维码**：CTA 区域的扫码入口，必须规划为一张图片 → seed: qrcode，尺寸通常 200x200
6. **美食/菜品**：餐饮类海报的菜品照片 → seed: food, dish, cuisine, dessert
7. **场地/环境**：活动场地、门店、酒店实景 → seed: venue, hotel, restaurant, interior
8. **品牌/Logo 衬托**：品牌形象相关的氛围图片 → seed: brand, corporate, luxury
9. **特性/卖点配图**：产品特性卡片的小图（比纯图标更有吸引力）→ seed: 与卖点相关
10. **作品集/案例**：展示过往作品或案例的多图网格 → seed: portfolio, gallery, artwork
11. **用户头像/评价**：客户口碑展示区的头像照片 → seed: avatar, customer, people

**seed 关键词选择规则：**
- 必须与海报主题直接相关，不要用 nature/abstract 等泛化词
- 食品海报 → food, coffee, bakery, restaurant, cooking
- 科技海报 → technology, laptop, smartphone, digital, coding
- 美妆海报 → beauty, cosmetics, skincare, makeup, perfume
- 运动海报 → fitness, running, gym, sports, yoga
- 旅行海报 → travel, beach, mountain, landmark, hotel
- 教育海报 → education, study, book, classroom, library
- 音乐海报 → music, concert, guitar, piano, headphones
- 商务海报 → business, office, meeting, professional, corporate
- 时尚海报 → fashion, clothing, style, dress, shoes
- 节日海报 → celebration, festival, party, gift, fireworks
- 如果是纯文字/几何设计，可以不用图片

## 图片蒙版约束（极其重要！）

**设计方案中如果使用了背景图片，必须明确蒙版策略：**

- 蒙版透明度**绝对不能超过 40%**，推荐 20%~30%
- 禁止在图片上使用不透明背景色作为蒙版（如 bg-gray-900、bg-red-600）
- 图片上方的内容区域不能有不透明背景色，否则图片会被完全遮挡
- 使用 textShadow 而非重蒙版来确保文字可读
- 在设计方案中写清楚：蒙版颜色和透明度（如 `bg-black/25`）

## 渲染引擎注意事项

本系统使用 DOM→Canvas 转换引擎，每个有背景色的 div 会变成独立矩形对象。
**背景色嵌套最多 3 层**（最外层 → 区块 → 卡片），超过 3 层嵌套会导致矩形互相遮挡。
设计方案中规划区块时请注意控制背景嵌套深度。

**禁止使用的 CSS 效果（引擎不支持，会导致视觉降级）：**

- `backdrop-blur`、`backdrop-filter` — 毛玻璃效果会完全丢失，变成纯色块
- CSS `filter`（blur、brightness 等）— 不会渲染
- `mix-blend-mode` — 不支持
- `clip-path` — 不支持

**替代方案：** 用更高透明度的纯色蒙版或渐变蒙版代替 backdrop-blur。

## 文字可读性规划（WCAG AA 对比度）

**在配色方案中，必须确保文字与背景的对比度满足可读性要求：**

- 深色背景（gray-800+）上的主文字：`text-white`（对比度 21:1），弱化文字不低于 `text-white/60`（约 9:1）
- 浅色背景（white/gray-50）上的主文字：`text-gray-800`（对比度 12:1），弱化文字不低于 `text-gray-500`（约 5:1）
- **禁止**在深色背景上使用 `text-white/30`（对比度仅 3:1，不可读）
- **禁止**在浅色背景上使用 `text-gray-300`（对比度仅 2:1，不可读）
- 图片上的文字区域必须规划 `textShadow`，不依赖纯蒙版

## 内容安全与合规

**设计方案的文案内容必须遵循以下底线：**

- 不创作涉及歧视、暴力、色情的内容
- 促销海报中的优惠规则必须清晰、不具有误导性
- 价格标注必须明确原价/折扣价的区别（原价使用 `line-through` 删除线）
- 如果用户要求的内容涉及敏感话题，用中性/正面的方式表达

## 负面提示（Negative Prompts — 绝对不要出现的设计问题）

**以下是常见的低质量设计特征，你的方案必须完全避免：**

1. **空旷海报**：只有一个大标题 + 大片空白，没有实质内容区块
2. **彩虹配色**：超过 3 个不相关的颜色同时出现，缺乏色彩纪律
3. **密度失衡**：所有内容挤在上半部分，下半部分空旷；或反之
4. **伪内容**：用"标题文字"、"描述内容"等占位符代替真实文案
5. **emoji 滥用**：用 emoji 或字母代替本该是图片的位置（亮点展示、人物头像）
6. **遮挡图片**：在背景图上叠加不透明色块（>40% 蒙版），图片完全看不见
7. **无焦点区块**：某个区块内没有明确的视觉焦点，元素均匀分布无主次
8. **字号单一**：全篇只有 1-2 个字号，缺乏排版层级（至少需要 4 级）
9. **断裂布局**：区块之间有明显的未定义空白间隙，视觉不连贯
10. **纯色平铺背景**：整个海报或大面积区块使用单一纯色（如 `bg-red-600`、`bg-blue-900`），缺乏层次和质感 → 必须用渐变、径向辉光、同色系深浅微变等高级手法
11. **直角容器**：卡片、按钮、标签使用直角矩形，缺乏现代感 → 所有容器必须有圆角（至少 `rounded-xl`）
12. **千篇一律的区块结构**：所有区块都是相同的"标题+文字"结构 → 必须混合使用卡片网格、时间轴、图文横排、引用框、数据展示等多种组件形式

## 搜索资料使用规范

**如果用户消息中包含了【联网搜索参考资料】，你必须：**

- 从搜索结果中提取**真实数据**（价格、时间、地点、活动名称、产品参数）融入设计方案的文案内容
- 使用搜索到的真实信息替代虚构内容（如真实的产品价格、真实的活动日期）
- 不要在设计方案中引用搜索来源 URL 或"参考 1"等标注
- 将搜索结果提炼为适合海报展示的精炼文案，不要直接复制原文
- 如果搜索结果包含多个数据来源且数值不一致，取最常见/最新的值

## 网格一致性规则

**设计方案中规划的所有区块必须遵循统一的空间网格：**

- 全部区块的左右内边距统一（推荐 `px-8` 即 32px），不要不同区块用不同的 padding
- 同一行的卡片宽度相等、间距统一（gap-4 或 gap-6）
- 区块间装饰线宽度统一（如都用 `w-24 h-0.5`）
- 在设计方案中明确标注统一参数：`统一 padding: px-8, 卡片间距: gap-4, 圆角: rounded-2xl`

## 注意事项

- **内容必须具体**：不要写"促销信息"，要写具体的金额、折扣、商品名
- **自行创作内容**：用户只给主题方向，你需要创作具体的促销文案、价格、规则等
- 所有文字内容使用中文
- 配色使用 Tailwind CSS 预定义颜色（如 red-500, yellow-400, blue-600）
- 区块之间要有**视觉节奏变化**（深色→浅色→深色交替，或同色系深浅变化+装饰线分割）
- 区块高度占比之和 = 100%，不能有大面积空白
- 图片 seed 关键词必须语义精准，与海报主题直接相关
- 每个区块必须有一个**视觉焦点**（大标题、核心数字、主图片）
- 排版参数必须与 Emotion 维度选择的情绪一致（字间距、圆角、阴影）
- gene.style 中的统一参数（borderStyle、cornerRadius、shadowLevel、tracking）必须贯穿所有区块
