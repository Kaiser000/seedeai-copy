# 产品简报详情包：Seede AI 复刻版

## 技术决策及理由

- **前端 React 而非 Vue**：最终选择 React，因为 LLM 生成 JSX 代码后可直接在前端渲染，技术栈统一
- **后端 Java（Spring Boot）**：用户偏好 Java，后端主要负责 LLM 调用转发、项目管理等，不参与渲染
- **LLM 选择 GLM-5**：OpenAI 协议兼容，成本可控；接口需设计为可配置，方便后续切换其他模型
- **Fabric.js 作为画布引擎**：确认采用 JSX → DOM → Fabric.js 路线，而非 DOM 直接编辑或混合方案
- **前端渲染而非后端 Puppeteer**：渲染在浏览器完成，不需要服务端无头浏览器，架构更简单
- **LLM 生成 JSX + Tailwind CSS**：而非纯 HTML 或 Vue 模板，LLM 对 React JSX 训练数据充足，生成质量较好

## 被排除的技术方案

- **生成纯 HTML + Tailwind**：更简单但缺少组件化结构，放弃
- **生成 Vue 模板**：LLM 训练数据覆盖不如 React，放弃
- **DOM 直接编辑（路线 A）**：拖拽/缩放手感不如 Canvas，导出质量可能有损，放弃
- **混合方案（路线 C）**：DOM 预览 + 交互覆盖层，复杂度高且定位模糊，放弃
- **后端 Puppeteer 渲染**：MVP 不需要，增加部署复杂度，放弃

## 核心技术风险

- **DOM → Fabric.js 转换**是最大难点：Tailwind CSS 样式（渐变、阴影、圆角、flex 布局、响应式类名）需逐一映射为 Fabric.js 对象属性。建议 MVP 先支持有限样式子集（纯色、基础文本、图片、简单边框）
- **Roll 单个元素的数据流**：需要把 Fabric.js 画布对象反向描述为 prompt 片段 → LLM 重新生成该元素的 JSX → 编译渲染 → 转换回 Fabric.js 对象替换原有对象。这条往返链路需要设计清楚
- **GLM-5 生成 JSX 质量未验证**：需要尽早做 prompt 测试，确认生成的 JSX 能正确渲染。如果质量不够，需要换模型或加强 System Prompt 约束
- **中文字体在 Canvas 上的渲染**：Fabric.js 文本对象的中文排版（换行、字间距、字体加载）可能需要额外处理

## LLM 流水线细节（源自 Seede AI 逆向分析）

- **阶段 1 - 需求分析**：LLM Chain-of-Thought，确定设计风格、配色基调、目标尺寸、内容结构
- **阶段 2 - 页面布局**：LLM 输出完整 JSX + Tailwind 代码，固定宽度 1080px，高度自适应
- **阶段 3 - 设计合成**：前端编译 JSX → 渲染 DOM → 解析 DOM 树生成图层树（文本/形状/图片节点）→ 转换为 Fabric.js 画布对象
- **阶段 4 - 图像处理**：导出时做压缩等后处理，超分等高级功能留后续
- 阶段 1 和 2 需要 LLM 参与（可合并为一次调用或拆两次），阶段 3 和 4 纯工程实现

## Seede AI 产品特征（从截图和文档提取）

- 首页支持场景选择：长图、活动海报、批量社媒图文、PPT、多页宣传册、易拉宝
- 输入方式：参考图片 + 素材 + 文档 + 文字描述
- 可选模型（截图中为 Gemini 3 Flash）、尺寸选择、图层数量控制
- 编辑器右侧面板：JSX 代码预览 + 页面布局树（形状/文本/图片分层）+ 设计合成/图像处理状态
- 编辑器底部：可继续用 AI 对话优化当前设计
- 工具栏：播放预览、添加元素、文字工具、图片工具、AI 功能入口

## 竞品调研摘要

- **稿定设计**：30万+模板，覆盖新媒体/电商/私域，¥8.3-39/月起
- **美图设计室**：AI 商品图、AI 海报，电商场景突出，¥20/月起
- **创客贴**：40万+模板，一键 AI 生成，¥79/季起
- **即梦 AI**（抖音旗下）：参考图+多风格生成，¥79/月
- **星流 AI**：自研大模型，中文字体适配好，批量生成，¥469/年
- **堆友 AI**（阿里系）：3D/水彩/扁平等多风格，¥299/年
- 行业趋势：AI 原生设计取代模板模式，中文排版是本土工具核心竞争力，电商场景是最大付费场景

## MVP 范围信号

- **明确 IN**：自然语言生成海报、Fabric.js 画布编辑（拖拽/缩放/文字编辑）、单元素 Roll、PNG 导出
- **明确 OUT**：多场景、用户系统、品牌配置、模板库、辅助工具（超分/背景移除）、商业化
- **用户态度**：先跑通流程，界面和交互模仿 Seede AI，不够好的地方做出来后体验改进

## 开放问题

- System Prompt 的具体设计：如何约束 LLM 生成的 JSX 结构，使其更容易被解析为 Fabric.js 对象？是否需要限制可用的 Tailwind 类名子集？
- Fabric.js 画布对象的数据模型设计：如何存储和序列化编辑状态？
- SSE 流式输出的前端展示：代码生成过程中用户看到什么？实时代码预览还是进度条？
- 多次 AI 对话的上下文管理：用户在编辑器中继续用 AI 优化设计时，如何传递当前画布状态给 LLM？
