## 二、代码模式分析（Prompt 规则校验）

### 2.1 规则遵循度矩阵

| 规则 | Prompt 要求 | 实际遵循率 | 差距评级 | 处理建议 |
|------|-----------|-----------|---------|---------|
| Tailwind 任意值 | 禁止使用 | **1.5%** 遵循 | **严重偏离** | 已放宽为"可用但推荐 inline style" |
| 图片 URL | picsum.photos | **0%** 遵循 | **完全不符** | 已更新为 placehold.co |
| 组件名称 | `function Poster()` | **0%** 遵循 | **完全不符** | 保持不变（不同渲染管线） |
| backdrop-blur | 禁止 | **47.2%** 遵循 | **大幅偏离** | 已更新为"可用" |
| overflow-hidden | 必须 | **95.4%** 遵循 | 良好 | 保持 |
| seede-root 类 | 未要求 | **100%** 使用 | — | 已在管线中处理 |
| JS 色彩对象 | 未要求 | **63.6%** 使用 | — | **已新增为必须模式** |
| textShadow | 必须 | **7%** 遵循 | 严重偏低 | 已在参考库中强化 |
| border-white/5 | 默认边框 | **8%** 遵循 | 严重偏低 | 调整为建议而非必须 |

### 2.2 样式实现方式

| 方式 | 使用率 | 典型用途 |
|------|--------|---------|
| Tailwind 类名（标准） | 100% | 布局（flex/grid）、间距（p/m/gap）、定位 |
| Tailwind 任意值（方括号） | 98.5% | 字号 text-[32px]、宽高 w-[1080px]、圆角 rounded-[40px] |
| inline style | 82.1% | 颜色 color/backgroundColor、字号 fontSize、字体 fontFamily |
| JS 色彩对象 | 63.6% | `const colors = { primary: '#...', ... }` |
| JS 排版对象 | 47% | `const fonts/typography = { h1: {...}, ... }` |

**结论：** 真实模板采用 **Tailwind 类名（布局） + inline style（视觉）** 的混合方案，其中高质量模板倾向于先定义 Token 对象再通过 inline style 引用。

### 2.3 图片使用模式

| 来源 | 使用次数 | 比例 |
|------|---------|------|
| placehold.co | 1,061 | 85% 模板 |
| static.seedeai.com | 58 | 12% 模板 |
| asset:// 协议 | 38 | 12% 模板 |
| picsum.photos | 0 | 0% |

- **94%** 模板使用 `<img>` 标签（符合规则）
- **80%** 模板在 `<img>` 上添加 `prompt="..."` 属性（英文图片描述，用于 AI 图片生成）
- **16%** 模板仍使用 `backgroundImage` inline style（轻度违反规则）

### 2.4 视觉效果使用率

| 效果 | 使用率 | 备注 |

## 附录：分析方法论

### 数据来源
- `backend/src/main/resources/all_items.json` — 325 个模板的完整数据
- 每个模板包含：id、name、description、prompt、sourceCode

### 分析工具
- Python 正则匹配提取代码模式
- 字符串搜索统计特征出现频率
- 分类算法基于关键词匹配

### 质量评分标准（满分 9 分）
- 标准 React 格式（+2）
- 定义 colors/theme 对象（+2）
- 定义 typography/fonts 对象（+2）
- 代码长度 8,000-20,000 字符（+1）
- 有意义的名称（+1）
- 有描述文字（+1）
