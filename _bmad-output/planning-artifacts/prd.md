# Product Requirements Document - Seede AI 复刻版

**Author:** Tfzhang11
**Date:** 2026-04-02

## 执行摘要

Seede AI 复刻版是一款面向信息型海报场景的 AI 设计工具，目标用户为缺乏专业设计能力但需要快速产出海报的社媒运营、营销人员和中小企业主。当前 AI 设计工具（豆包、即梦等）基于图片大模型生成扁平图片，用户无法精细编辑——修改一个字就需要整体重新生成，结果不可控且成本高。本产品采用"代码即设计"路线，利用 LLM 代码生成能力的可控性优势，将自然语言描述转化为可逐元素编辑的画布设计，让 AI 生成的设计像 PPT 一样可控。

### 核心差异化

**以强补弱**：LLM 的 vibe coding 能力远强于图片生成能力。本产品利用这一不对称优势，通过 LLM 生成 JSX + Tailwind CSS 结构化代码，前端渲染为 DOM 后转换为 Fabric.js 画布对象，使每个文本、形状、图片都成为独立可编辑图层。用户不再"摇骰子"——生成不满意时可直接修改具体元素，或对单个元素调用 AI 重新生成（Roll），把运气问题变成手艺问题。

核心壁垒在于 System Prompt 的设计质量（决定生成的配色、排版、视觉层次）以及 DOM → Fabric.js 转换引擎的成熟度。同时具备成本优势（代码生成远低于图片生成的 token 消耗）、文字精确可控（文字永远不会变形或模糊）、以及未来品牌一致性扩展的潜力。

需注意：图片大模型的局部编辑能力正在快速进步，当前可编辑性优势存在时间窗口，MVP 需快速验证核心流程。

## 项目分类

- **项目类型**：Web App + AI Pipeline（React MPA 前端 + Spring Boot 后端 LLM 编排层）
- **领域**：AI 辅助设计 / 内容生成
- **复杂度**：中高（核心技术风险集中在 DOM→Fabric.js 转换和 LLM 生成质量，无合规/监管约束）
- **项目上下文**：绿地项目（全新开发）
- **资源配置**：单人全栈开发

## 成功标准

### 用户成功

- 非专业设计用户从输入描述到导出可用海报，全流程 **5 分钟以内**完成
- 用户能在 AI 生成的基础上发挥个人创意，而非只能被动接受结果
- 用户遇到修改需求（如领导要求改字）时，可直接在画布上编辑，**无需重新生成**

### 业务成功

MVP 阶段定位为**技术验证**，非商业化阶段：

- 核心技术链路（自然语言 → JSX 生成 → DOM 渲染 → Fabric.js 画布 → 编辑 → 导出）完整跑通
- 自己用着满意——流程流畅、效果达到"能用"水平
- 验证"代码即设计"路线的可行性，为后续迭代和外部用户试用奠定基础

### 技术成功

- **DOM → Fabric.js 转换**：基础样式（纯色、文本、图片、简单边框）正确转换，无明显变形
- **LLM 生成质量**：GLM-5 生成的 JSX 能正确编译渲染，无需人工修代码
- **Roll 单元素**：选中元素 → AI 重新生成 → 替换回画布的完整闭环可用
- **中文字体**：Canvas 上中文文字显示正常，无乱码或排版错乱

### 可衡量成果

- 首次生成的海报"看起来正常"（布局合理、文字不重叠、配色不刺眼、内容结构完整）
- 用户通过对话或手动编辑后，能产出满意的最终成品
- 全流程无阻断性 bug，用户不需要刷新页面或重新开始

## 用户旅程

### 旅程 1：运营人员的高效交付（成功路径）

**角色**：小美，互联网公司社媒运营，非设计专业背景

**背景**：领导让她在明天之前做一张五一促销活动海报。她先用豆包生成了几版，效果还行，但领导要求"标题再大一点，价格用红色"。她重新生成了三次，布局全变了，越改越差，半小时过去了还没搞定。

**发现产品**：小美打开 Seede AI，在输入框里写："五一劳动节促销海报，主标题'五一狂欢购'，副标题'全场低至5折'，底部放活动时间5月1日-5月3日，风格简约现代，蓝白配色"。

**生成与编辑**：30 秒后，画布上出现了一张完整海报。布局合理，配色清爽——虽然不完美，但已经是一张"正常的海报"。她直接双击标题调大字号，选中价格文字改成红色。领导又说"背景能不能换个感觉"，她选中背景元素点了 Roll，AI 生成了三个方案，她选了第二个。

**交付**：点击导出，拿到 PNG，发给领导确认通过。全程 4 分钟。

**情感弧线**：焦虑（原工具反复失败）→ 好奇（试试新工具）→ 惊喜（一键生成完整海报）→ 满足（精细调整后快速交付）

### 旅程 2：小店老板的挫折与恢复（边缘场景）

**角色**：小李，奶茶店老板，零设计经验

**背景**：想做一张新品上市海报发朋友圈推广，输入："奶茶新品上市，名字叫桃桃乌龙，粉色系，少女风"。

**遇到问题**：AI 首次生成的布局不理想——文字和装饰元素挤在一起，背景色偏深。

**恢复路径**：小李在对话框里追加："文字之间间距大一点，背景换成浅粉色"。AI 根据当前画布状态重新调整，这次布局舒服多了。他手动把产品名拖到更居中的位置，选中价格文字 Roll 了一次换了个更活泼的样式。

**交付**：导出 PNG，发到朋友圈。虽然比预期多花了两分钟，但最终满意。

**情感弧线**：期待（新工具试试）→ 失望（首次生成不够好）→ 尝试（对话调整 + 手动编辑）→ 满意（最终效果达标）

### 旅程需求总结

| AI 生成 | 首次生成达到"能看"水平 | 生成质量有波动时不致命 |
| 画布编辑 | 双击编辑文字、调整样式 | 拖拽调整位置 |

## 创新与新颖模式

### 创新领域

**"代码即设计"范式**：区别于市场上所有基于图片大模型的 AI 设计工具，本产品采用 LLM 生成结构化代码（JSX + Tailwind CSS）作为设计的中间表达。这一路线已被 Seede AI 验证可行，核心创新在于：

- 利用 LLM 代码生成能力远强于图片生成能力的不对称优势（以强补弱）
- 结构化输出天然支持元素级编辑，解决了图片大模型"不可编辑"的根本缺陷
- 成本显著低于图片生成（代码 token 消耗远低于图像生成）

**JSX → DOM → Fabric.js 转换管线**：将前端代码渲染技术与画布编辑引擎结合，使 AI 生成的代码无缝转化为用户可操作的画布对象。这一管线在设计工具领域是新颖的工程方案。

### 市场背景与验证

该路线已被 Seede AI 产品验证可行（支持多场景设计生成 + 画布编辑）。本项目是在其验证基础上的工程复现，聚焦海报单一场景做深做透。无需 POC 阶段，直接构建完整 MVP。验证重点在工程实现质量：DOM → Fabric.js 转换的保真度、LLM 生成 JSX 的可渲染率。

## Web App + AI Pipeline 特定需求

### 项目类型概览

多页应用（MPA）架构，包含两个核心页面：

- **输入页**：用户输入自然语言描述，发起海报生成请求
- **编辑器页**：分栏布局——左侧展示 LLM 流式输出过程，右侧为 Fabric.js 画布编辑器

### 技术架构考量

**前端架构**：

- React MPA，页面间通过路由切换
- 输入页：文本输入 + 生成按钮，交互简单
- 编辑器页：左右分栏——左侧 SSE 流式内容展示（代码/状态），右侧 Fabric.js 画布
- Tailwind CSS 负责 UI 样式 + 作为 LLM 生成海报代码的样式方案

**后端架构（Spring Boot）**：

- LLM 编排层：接收用户输入 → 构造 System Prompt → 调用 GLM-5 → SSE 流式返回
- LLM 接口设计为可配置，支持后续切换模型
- OpenAI 协议兼容接口，降低模型切换成本

**流式通信**：

- SSE（Server-Sent Events）从后端推送 LLM 生成过程到前端
- 前端实时展示代码生成进度，生成完成后触发 DOM 渲染 → Fabric.js 转换

### 页面流转

```text
输入页（自然语言描述）
  ↓ 提交
编辑器页
  ├── 左栏：SSE 流式展示 LLM 输出（JSX 代码 / 生成状态）
  └── 右栏：Fabric.js 画布（生成完成后渲染，支持编辑）
       ↓ 用户编辑（拖拽/改文字/Roll/对话优化）
       ↓ 导出 PNG
```

### 实现考量

- **浏览器兼容性**：标准现代浏览器支持，无特殊兼容要求
- **SEO**：不需要
- **无障碍**：MVP 不考虑
- **响应式设计**：编辑器页固定桌面端布局，输入页可适当响应式
- **状态管理**：编辑器页需管理画布状态（Fabric.js 对象树）、对话历史、SSE 连接状态

## 产品范围与分阶段开发

### MVP 策略

**MVP 类型**：问题解决型——验证"代码即设计"路线能解决"AI 生成不可编辑"的核心问题。
**核心原则**：流程跑通优先，不追求用户量或商业指标。

**支持的核心用户旅程**：

- 旅程 1（成功路径）：自然语言 → 生成 → 编辑 → 导出，5 分钟内完成
- 旅程 2（边缘场景）：首次生成不理想 → 对话优化 + 手动编辑 → 恢复

### MVP 功能集（Phase 1）

- 自然语言输入生成海报（LLM 需求分析 + JSX 代码生成）
- JSX → DOM → Fabric.js 画布渲染（基础样式子集）
- 画布编辑：选中、拖拽、缩放、删除元素、添加文字
- 文字内容编辑与基础样式修改（颜色、字号）
- 撤销/重做
- 单元素 AI 重新生成（Roll）
- 对话式 AI 优化当前设计
- 海报尺寸选择
- SSE 流式输出展示生成过程
- PNG 导出（所见即所得，不低于 1080px 宽度）

### Phase 2（成长）

- 多场景支持（长图、社媒图文、PPT）
- 扩展 Tailwind → Fabric.js 样式映射（渐变、阴影、圆角）
- 素材库 / 图片搜索集成
- 用户系统与历史记录
- 品牌配置（锁定配色、字体规范）

### Phase 3（扩展）

- 全场景覆盖（电商详情页、易拉宝、宣传册）
- 参考图片输入 + 风格迁移
- 批量生成
- 辅助工具（超分 / 背景移除）
- 商业化（付费、订阅）
- 接入更强 LLM 持续提升生成质量

### 风险缓解策略

- **LLM 生成质量不足**：接口可配置，随时切换更强模型
- **DOM → Fabric.js 转换丢失信息**：MVP 先支持有限样式子集，逐步扩展；转换流程可持续改造
- **时间窗口风险**：图片大模型局部编辑能力在快速进步，需快速交付 MVP 抢占先机
- **单人开发资源风险**：严格控制 MVP 范围，不扩展功能直到核心流程完全跑通

## 功能需求

### 海报生成

- FR1: 用户可以通过自然语言描述输入海报需求（包括主题、内容、风格、配色等）
- FR2: 系统可以将用户输入发送至 LLM 进行需求分析和代码生成
- FR3: 系统可以通过 SSE 流式输出实时展示 LLM 生成过程
- FR4: 系统可以将 LLM 生成的 JSX + Tailwind CSS 代码编译渲染为 DOM

### DOM 到画布转换

- FR5: 系统可以解析渲染后的 DOM 树，识别文本、形状、图片等元素节点
- FR6: 系统可以将 DOM 元素转换为对应的 Fabric.js 画布对象（支持基础样式子集：纯色、文本、图片、简单边框）
- FR7: 系统可以在画布上保持 DOM 渲染时的布局和层级关系

### 画布编辑

- FR8: 用户可以在画布上选中任意元素
- FR9: 用户可以拖拽元素调整位置
- FR10: 用户可以缩放元素调整大小
- FR11: 用户可以双击文字元素进行内容编辑
- FR12: 用户可以修改文字的基础样式（颜色、字号）
- FR13: 用户可以撤销和重做画布上的编辑操作
- FR14: 用户可以选择或查看海报的尺寸规格
- FR15: 用户可以删除画布上的选中元素
- FR16: 用户可以在画布上添加新的文字元素

### AI 交互优化

- FR17: 用户可以对选中的单个元素触发 AI 重新生成（Roll），获得替代方案
- FR18: 用户可以通过对话框向 AI 描述修改意图，AI 基于当前画布状态生成调整后的完整设计并替换画布
- FR19: 系统可以将当前画布的结构化描述（元素类型、位置、内容、样式）序列化为 LLM 可理解的上下文格式

### 导出

- FR20: 用户可以将画布内容一键导出为 PNG 图片

### LLM 配置

- FR21: 系统支持可配置的 LLM 接口（OpenAI 协议兼容），可切换不同模型

## 非功能需求

### 性能

- LLM 生成过程采用 SSE 流式输出，用户在流式输出期间不视为等待，无响应时间硬性要求
- 画布编辑操作（拖拽、缩放、选中、文字编辑）应尽量流畅，无明显卡顿
- 撤销/重做操作应即时响应

### 导出质量

- 导出的 PNG 图片与画布上所见一致（所见即所得），分辨率不低于 1080px 宽度
- 导出图片色彩、文字、布局不应与画布显示存在偏差

### 集成

- 后端通过 OpenAI 协议兼容接口与 LLM（GLM-5）集成
- LLM 接口配置可切换，支持后续更换模型而无需修改业务代码
- SSE 作为前后端流式通信协议

### 中文支持

- Fabric.js 画布上中文文字显示正常，无乱码
- 中文文本换行、字间距渲染正确
- 需确保中文字体正确加载到 Canvas 环境
